# ADR-003: Разделение `.env` для локальной и docker-среды

## Контекст
В проекте есть два режима работы: локальные Python-скрипты запускаются на `localhost`, а контейнеры в Docker-сети должны обращаться друг к другу по именам сервисов. Один общий файл окружения для этих сценариев быстро приводит к ручным правкам и ошибкам в адресах.

## Решение
Использовать два файла окружения: `.env.local` для запуска скриптов вне контейнеров и `.env.docker` для сервисов в compose-сети. Выбор окружения централизован в `config/settings.py`, который подгружает нужный файл по `APP_ENV`.

## Последствия
Локальный запуск и запуск через `docker compose` больше не мешают друг другу, а адреса сервисов остаются корректными для каждого режима. При этом шаблоны `.env.local.example` и `.env.docker.example` нужно держать синхронно, чтобы конфигурация не расходилась.
