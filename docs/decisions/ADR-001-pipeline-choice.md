# ADR-001: Выбор конвейера слоёв

## Контекст
Проект строит локальный lakehouse поверх датасета H&M и уже использует отдельные слои для raw, bronze, silver и mart. Для команды важно, чтобы путь от исходных CSV до BI-витрин был простым, повторяемым и совпадал с текущими скриптами загрузки.

## Решение
Зафиксировать последовательность `raw -> bronze -> silver -> marts`. Raw хранит исходники без бизнес-трансформаций, bronze содержит технически нормализованные Iceberg-таблицы, silver хранит очищенную аналитическую модель, а marts служат стабильным слоем для Trino и BI-агента.

## Последствия
Такой конвейер прозрачен для демо и отладки, а витрины проще поддерживать и переиспользовать в семантическом слое. Цена решения в том, что нужно сопровождать несколько слоёв и обновлять их последовательно, а не в одном месте.
