# ADR-004: Единый класс настроек Python

## Контекст
Python-скрипты проекта работают и локально, и внутри Docker-сети, поэтому им нужен единый способ читать переменные окружения, собирать URI слоёв и переиспользовать адреса сервисов. Сейчас эта роль уже отдана модулю `config/settings.py`.

## Решение
Хранить базовые параметры проекта в неизменяемом `Settings`-классе и создавать его один раз через `Settings.from_env()`. Такой подход убирает дублирование строк с переменными окружения и даёт общий источник правды для ingestion-скриптов и вспомогательных модулей.

## Последствия
Код становится проще сопровождать и тестировать: новые параметры добавляются в одно место, а вычисляемые URI собираются через свойства класса. Обратная сторона в том, что при вводе новой переменной окружения нужно не забыть обновить `config/settings.py` и связанные `.env`-шаблоны.
