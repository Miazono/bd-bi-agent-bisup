# PDR-002: Размещение raw-файлов

## Контекст
Hive external tables и Bronze-загрузка ожидают, что файлы одного источника будут лежать в отдельном каталоге. Если смешать разные датасеты или партии в одном raw-префиксе, становится сложнее повторять загрузку и проверять состав батча.

## Решение
Хранить файлы по схеме `raw/hm/<источник>/load_date=YYYY-MM-DD/<filename>`. Для текущего проекта это `raw/hm/articles/...`, `raw/hm/customers/...` и `raw/hm/transactions_train/...`, что совпадает с `ingestion/load_raw.py` и загрузкой внешних raw-таблиц.

## Последствия
Такой layout не смешивает разные источники и даёт стабильный путь для повторных запусков по дате загрузки. Bronze-скрипт может однозначно найти нужный файл, а Hive/Trino не будут склеивать несовместимые CSV в одну таблицу.
