# Задача проекта

## Цель
Собрать локальный Data Lakehouse и BI-агента поверх реальных витрин данных, не отходя от текущего стека репозитория.

## Что входит в задачу
- загрузить исходные CSV в `MinIO` в raw-слой;
- построить цепочку `raw -> bronze -> silver -> marts` на базе `Iceberg` и `Trino`;
- подготовить набор аналитических витрин для вопросов по данным;
- подключить BI-агента `WrenAI` или совместимое решение к витринам;
- показать рабочий сценарий вопросов на естественном языке и SQL-ответов по данным.

## В рамках проекта
- основной датасет: H&M Fashion Recommendations;
- v1 работает только с табличными файлами `articles.csv`, `customers.csv`, `transactions_train.csv`;
- изображения и внешние источники в текущем scope не используются;
- акцент делается на воспроизводимую локальную сборку, а не на облачную инфраструктуру.

## Ожидаемый результат
- поднятый локальный стек `MinIO + Hive Metastore + Iceberg + Trino`;
- загруженные и обработанные данные в lakehouse-слоях;
- готовые marts для аналитики;
- BI-агент, который опирается на реальные витрины и семантический слой;
- короткое демо с ответами на пользовательские вопросы и базовой проверкой качества.
