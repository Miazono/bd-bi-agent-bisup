# Обзор стека

## Назначение
Этот проект собирает локальный учебный lakehouse и слой BI-агента вокруг датасета H&M.

## Основные компоненты
- `MinIO` хранит raw-файлы и физические данные Iceberg-таблиц.
- `Hive Metastore` выступает каталогом таблиц для `Iceberg`.
- `Trino` используется как SQL-движок для чтения и наполнения слоёв.
- `WrenAI` подключается отдельным compose-профилем `wrenai` и работает поверх `Trino`.
- `Qdrant`, `wren-engine`, `wren-ui` и `ibis-server` входят в BI-часть стека.
- `Python` отвечает за инициализацию хранилища и ETL/ELT-скрипты.

## Слои данных
- `raw` хранит исходные CSV без бизнес-трансформаций.
- `bronze` содержит технически нормализованные Iceberg-таблицы.
- `silver` хранит очищенную аналитическую модель.
- `marts` содержит витрины, удобные для BI и семантического слоя.

## Конфигурация запуска
- локальные скрипты используют `.env.local`;
- контейнерный запуск использует `.env.docker`;
- выбор окружения централизован в `config/settings.py`;
- инфраструктура поднимается через `docker-compose.yml`, а сервисы BI-агента включаются отдельным профилем.

## Практический смысл
Такой стек даёт воспроизводимый путь от сырых файлов до готовых витрин и позволяет подключать BI-агента к стабильному SQL-слою без доступа к исходным данным.
