# Инженерия промптов

## Цель
Сделать SQL-агента предсказуемым: он должен отвечать только на базе доступных витрин, корректно выбирать метрики и не смешивать бизнес-термины с техническими деталями.

## Что уже есть в репозитории
- `bi-agent/prompts/system_prompt.md` — место для системных правил агента;
- `bi-agent/semantic_layer/*.yaml` — семантический слой с моделями, связями и метриками;
- `bi-agent/eval/questions.json` — набор контрольных вопросов для проверки качества;
- `bi-agent/eval/run_eval.py` — базовый прогон вопросов к BI-агенту;
- `infra/wrenai/config.yaml` — конфигурация WrenAI, LLM и конвейеров поиска контекста.

## Принципы формулировки промптов
- агент должен опираться на `mart`-витрины и их описание, а не на raw-данные;
- в ответах важно использовать те же названия метрик и измерений, что и в семантическом слое;
- если вопрос неоднозначен, лучше запросить уточнение, чем строить случайный SQL;
- при генерации SQL приоритет у читаемости и соответствия схеме, а не у “красивого” текста;
- промпты должны оставаться короткими и опираться на явные правила, а не на длинные общие инструкции.

## Как улучшать качество
- добавлять новые вопросы в `bi-agent/eval/questions.json` вместе с ожидаемыми колонками;
- обновлять описание витрин, если меняется бизнес-логика;
- проверять, что SQL по-прежнему обращается к нужной mart-таблице;
- фиксировать удачные шаблоны запросов и типовые ошибки агента;
- при необходимости расширять системный промпт правилами по стилю ответа и ограничениями по источникам данных.
