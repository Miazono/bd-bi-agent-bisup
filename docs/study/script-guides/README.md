# Аннотированные копии скриптов и быстрые схемы

## Назначение

Этот раздел дополняет обычные разборные документы из `docs/study/project/`.
Здесь материалы сделаны в другом формате:

- `annotated/` — учебные копии исходных скриптов с подробными русскими комментариями прямо в коде;
- `maps/` — короткие схемы для быстрого чтения: что делает файл, какие у него входы, какие функции есть внутри и каков порядок выполнения.

## Зачем нужен отдельный формат

Обычные документы удобны, когда нужно последовательно изучить тему и понять общий замысел.
Этот раздел нужен для двух других сценариев:

- быстро открыть почти тот же код и читать его с пояснениями построчно;
- быстро вспомнить структуру файла, не перечитывая длинный текстовый разбор.

## Как пользоваться разделом

Для каждого важного скрипта лучше читать материалы парой:

1. Сначала открыть краткую схему из `maps/`.
2. Потом открыть аннотированную копию из `annotated/`.
3. При необходимости свериться с полным разбором из `docs/study/project/`.

Такой порядок даёт три уровня детализации:

- очень краткий;
- почти по коду;
- полнотекстовый разбор.

## Структура

Раздел повторяет логику исходного репозитория:

- `annotated/ingestion/` — копии `load_raw.py`, `load_bronze.py`, `load_silver.py`, `load_marts.py`;
- `annotated/scripts/` — копии вспомогательных скриптов;
- `maps/ingestion/` — быстрые схемы по загрузочным скриптам;
- `maps/scripts/` — быстрые схемы по вспомогательным скриптам.

## Важное правило чтения

Аннотированные копии нужны для обучения и разбора.
Источником истины всё равно остаются реальные файлы из:

- `ingestion/`
- `scripts/`

Если аннотированная копия и исходник расходятся, верить нужно исходнику и фактическому коду репозитория.
